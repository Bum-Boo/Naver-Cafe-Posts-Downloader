# 네이버 카페 아카이브 매니저

네이버 카페 게시글을 내 PC에 보관하고, 나중에 오프라인으로 다시 열어볼 수 있게 만드는 Python 데스크톱 앱입니다.

이 앱의 목표는 네이버 카페에서 내가 로그인한 상태로 볼 수 있는 게시글을 `content.txt`, `content.html`, `view.html`, 이미지 파일, 메타데이터로 정리해 저장하는 것입니다. 네이버 블로그용 도구가 아니며, 네이버 카페 게시글과 메뉴/게시판 주소만 대상으로 합니다.

## 중요한 원칙

- 네이버 카페 전용 앱입니다. 네이버 블로그 로직을 사용하지 않습니다.
- 사용자가 직접 로그인한 네이버 세션으로 볼 수 있는 글만 저장합니다.
- 카페 가입, 게시판 권한, 비공개 글 제한을 우회하지 않습니다.
- 네이버 로그인은 사용자가 직접 브라우저에서 진행합니다.
- 비밀번호를 저장하지 않습니다.
- 로그인 세션 파일과 브라우저 프로필은 로컬 데이터이며 git에 올리지 않습니다.

## 주요 기능

- 개별 네이버 카페 게시글 다운로드
- 메뉴/게시판 주소에 보이는 게시글 일괄 다운로드
- 저장된 글 목록을 개별 다운로드와 메뉴 다운로드로 구분해서 보기
- 저장된 글의 제목, 원본 주소, 저장일, 이미지 수, 폴더 경로 확인
- 저장된 `view.html`을 기본 브라우저로 열기
- 저장 폴더 열기
- 게시글 폴더명 또는 메뉴 상위 폴더명 변경
- 저장 항목 삭제
- 다운로드 진행 팝업, 실시간 로그, 취소 버튼
- 로그인 세션 상태 확인

## 설치

Python 3.11 이상을 권장합니다.

1. 프로젝트 폴더로 이동합니다.

```bash
cd "Naver Cafe post downloader"
```

2. 필요한 Python 패키지를 설치합니다.

```bash
pip install -r requirements.txt
```

3. Playwright Chromium을 설치합니다.

```bash
python -m playwright install chromium
```

## 실행

```bash
python app.py
```

앱 창이 열리면 상단에 다운로드 모드, 주소 입력칸, 다운로드 버튼, 네이버 로그인 세션 연결 버튼, 세션 상태 표시가 보입니다.

## Windows 실행 파일로 배포하기

명령어 없이 아이콘을 더블클릭해서 실행되는 Windows 앱 형태로 만들 수 있습니다. 이 프로젝트는 PyInstaller 기반 배포 설정을 포함합니다.

빌드 전 준비:

1. 앱 아이콘을 바꾸고 싶으면 `assets/app_icon.ico` 파일을 넣습니다.
2. 아이콘 파일을 넣지 않아도 빌드는 됩니다. 이 경우 기본 실행 파일 아이콘이 사용됩니다.
3. 아이콘을 바꾼 뒤에는 다시 빌드해야 새 실행 파일에 반영됩니다.

빌드 방법:

```powershell
powershell -ExecutionPolicy Bypass -File packaging/windows/build_portable.ps1
```

빌드 스크립트가 하는 일:

1. `requirements.txt`의 실행 의존성을 설치합니다.
2. Playwright Chromium을 포터블 패키지용 `ms-playwright/` 폴더에 준비합니다.
3. `NaverCafeArchiveManager.spec` 설정으로 onedir 실행 파일을 생성합니다.
4. `README_QUICKSTART.txt`를 배포 폴더에 복사합니다.
5. 빈 `data/`, `saved_posts/` 런타임 폴더를 만듭니다.
6. `release/NaverCafeArchiveManager-v0.1.0-win64.zip` 파일을 생성합니다.

빌드 결과:

```text
release/
  NaverCafeArchiveManager-v0.1.0-win64.zip
```

ZIP 내부 구조:

```text
NaverCafeArchiveManager/
    NaverCafeArchiveManager.exe
    _internal/
    ms-playwright/
    README_QUICKSTART.txt
    data/
    saved_posts/
```

배포할 때는 `release/NaverCafeArchiveManager-v0.1.0-win64.zip` 파일을 전달하세요. 사용자는 압축을 푼 뒤 `NaverCafeArchiveManager.exe`를 더블클릭하면 됩니다.

주의할 점:

- `NaverCafeArchiveManager.exe`만 따로 복사하지 말고 ZIP 내부의 `NaverCafeArchiveManager/` 폴더 전체를 함께 사용해야 합니다.
- 첫 실행 후 로그인 세션과 저장 글은 실행 파일 옆의 `data/`, `saved_posts/`에 생성됩니다.
- 다른 PC로 배포하면 해당 PC에서 네이버 로그인 세션을 다시 연결해야 합니다.
- `data/auth/`, `data/browser_profile/`, `saved_posts/`는 개인 데이터이므로 배포용 압축 파일에 기존 사용자의 데이터를 넣지 마세요.

## 처음 사용하는 순서

1. `python app.py`로 앱을 실행합니다.
2. `네이버 로그인 세션 연결` 버튼을 누릅니다.
3. Chromium 브라우저가 열리면 네이버에 직접 로그인합니다.
4. 앱으로 돌아와 로그인 세션 연결이 완료될 때까지 기다립니다.
5. 상단 세션 상태가 `적용됨`으로 표시되는지 확인합니다.
6. 다운로드 모드를 선택합니다.
7. 네이버 카페 게시글 주소 또는 메뉴/게시판 주소를 붙여넣습니다.
8. `다운로드` 버튼을 누르거나 주소 입력칸에서 Enter를 누릅니다.
9. 진행 팝업에서 현재 단계와 로그를 확인합니다.
10. 완료 후 왼쪽 저장 목록에서 저장된 글을 선택합니다.
11. `저장 페이지 열기`를 눌러 저장된 `view.html`을 엽니다.

## 로그인 세션 연결

이 앱은 Playwright Chromium의 앱 전용 프로필을 사용합니다.

- 브라우저 프로필: `data/browser_profile/`
- 보조 인증 상태: `data/auth/naver_state.json`

처음 한 번 로그인해 두면 이후 다운로드에서 같은 세션을 재사용합니다. 세션이 만료되었거나 카페 접근 권한 확인이 필요한 경우 앱이 아래 메시지를 표시합니다.

```text
네이버 로그인 세션이 만료되었거나 카페 접근 권한 확인이 필요합니다. [네이버 로그인 세션 연결]을 다시 실행해주세요.
```

이 메시지가 나오면 `네이버 로그인 세션 연결` 버튼을 다시 눌러 로그인 상태를 갱신하세요.

## 다운로드 모드 선택

상단의 다운로드 모드에서 세 가지 중 하나를 선택합니다.

- `자동 감지`: 주소 형식을 보고 개별 게시글 또는 메뉴 전체 다운로드로 자동 실행합니다.
- `개별 게시글 다운로드`: 게시글 주소만 허용합니다. 메뉴 주소를 넣으면 안내 메시지가 표시됩니다.
- `메뉴 전체 다운로드`: 메뉴/게시판 주소만 허용합니다. 게시글 주소를 넣으면 안내 메시지가 표시됩니다.

처음에는 `자동 감지`를 사용하면 됩니다. 주소 종류를 명확히 분리하고 싶을 때만 수동 모드를 선택하세요.

## 개별 게시글 다운로드

개별 게시글 하나만 저장할 때 사용합니다.

지원하는 주소 형식:

- `https://cafe.naver.com/{cafeName}/{articleId}`
- `https://cafe.naver.com/ArticleRead.nhn?clubid={clubId}&articleid={articleId}`
- `https://cafe.naver.com/ca-fe/cafes/{clubId}/articles/{articleId}`
- `https://m.cafe.naver.com/ca-fe/web/cafes/{clubId}/articles/{articleId}`

사용 순서:

1. 다운로드 모드를 `자동 감지` 또는 `개별 게시글 다운로드`로 선택합니다.
2. 주소 입력칸에 네이버 카페 게시글 주소를 붙여넣습니다.
3. `다운로드` 버튼을 누르거나 Enter를 누릅니다.
4. 같은 게시글이 이미 저장되어 있으면 다시 다운로드할지 확인하는 창이 뜹니다.
5. 같은 이름의 폴더가 이미 있으면 새 폴더를 만들지 확인하는 창이 뜹니다.
6. 완료되면 왼쪽 목록의 `개별 다운로드` 아래에 저장된 글이 추가됩니다.

저장 구조:

```text
saved_posts/
  게시글제목/
    content.txt
    content.html
    view.html
    meta.json
    images/
      001.jpg
      002.png
```

저장되는 파일:

- `content.txt`: 게시글 본문 텍스트
- `content.html`: 게시글 본문 HTML
- `view.html`: 로컬에서 보기 좋은 형태로 재구성한 HTML
- `meta.json`: 원본 주소, 제목, 저장일, 이미지 목록, iframe 사용 여부 등 메타데이터
- `images/`: 게시글 본문 안에서 수집한 이미지

## 메뉴/게시판 전체 다운로드

메뉴 또는 게시판에 보이는 여러 게시글을 한 번에 저장할 때 사용합니다.

지원하는 메뉴 주소 형식:

- `https://cafe.naver.com/f-e/cafes/{clubId}/menus/{menuId}`
- `https://cafe.naver.com/ca-fe/cafes/{clubId}/menus/{menuId}`
- `https://m.cafe.naver.com/ca-fe/web/cafes/{clubId}/menus/{menuId}`

예시:

- `https://cafe.naver.com/f-e/cafes/30636006/menus/24`
- `https://cafe.naver.com/f-e/cafes/30636006/menus/65`

사용 순서:

1. 다운로드 모드를 `자동 감지` 또는 `메뉴 전체 다운로드`로 선택합니다.
2. 주소 입력칸에 네이버 카페 메뉴/게시판 주소를 붙여넣습니다.
3. `다운로드` 버튼을 누르거나 Enter를 누릅니다.
4. 앱이 메뉴 페이지에서 현재 로그인 세션으로 보이는 게시글 링크를 수집합니다.
5. 수집이 끝나면 게시글을 하나씩 순차 다운로드합니다.
6. 이미 저장된 게시글은 `article_key` 기준으로 자동 스킵합니다.
7. 완료되면 왼쪽 목록의 `메뉴 다운로드` 아래에 메뉴 그룹과 게시글이 추가됩니다.

메뉴 일괄 저장 구조:

```text
saved_posts/
  menus/
    24_게시판제목/
      게시글제목/
        content.txt
        content.html
        view.html
        meta.json
        images/
          001.jpg
```

배치 실행 로그:

- `data/batches/{batch_id}.json`: 다운로드/스킵/실패 요약
- `data/batches/{batch_id}_failed_urls.txt`: 실패한 게시글 주소 목록

메뉴 다운로드는 공격적인 요청을 피하기 위해 게시글을 병렬로 받지 않고 순차적으로 처리합니다. 다운로드 도중 취소하면 이미 완료된 게시글은 그대로 남고, 배치 로그에 취소 시점과 남은 개수가 기록됩니다.

## 진행 팝업과 취소

다운로드를 시작하면 진행 팝업이 열립니다.

팝업에서 확인할 수 있는 내용:

- 현재 실행 단계
- 전체 개수가 확인된 경우 진행률
- 게시글 또는 이미지 처리 로그
- 실패한 주소 또는 스킵된 주소

`취소` 버튼을 누르면 안전한 지점에서 다운로드가 중단됩니다. 강제로 스레드를 종료하지 않기 때문에 이미 저장이 끝난 게시글과 파일은 유지됩니다.

## 저장된 글 보기와 관리

왼쪽 저장 목록은 트리 구조입니다.

- `개별 다운로드`: 개별 게시글 다운로드로 저장한 글
- `메뉴 다운로드`: 메뉴/게시판 전체 다운로드로 저장한 글

목록 위의 `목록 필터`에서 아래 보기 방식을 선택할 수 있습니다.

- `전체 보기`
- `개별 다운로드만`
- `메뉴 다운로드만`

게시글을 선택하면 오른쪽에서 아래 정보를 확인할 수 있습니다.

- 제목
- 원본 주소
- 저장일
- 이미지 수
- 저장 폴더 경로
- 다운로드 타입
- 메뉴 제목, 메뉴 ID, 배치 ID
- 미리보기

버튼 사용법:

- `저장 페이지 열기`: 선택한 게시글의 `view.html`을 기본 브라우저에서 엽니다.
- `저장 폴더 열기`: 선택한 게시글 폴더 또는 메뉴 상위 폴더를 엽니다.
- `폴더명 변경`: 선택한 게시글 폴더명 또는 메뉴 상위 폴더명을 변경합니다.
- `저장 항목 삭제`: 선택한 게시글 또는 메뉴 그룹을 삭제합니다.
- `목록 새로고침`: 저장 목록을 다시 불러옵니다.

메뉴 그룹을 선택한 경우 `저장 페이지 열기`는 비활성화됩니다. 대신 메뉴 제목, 메뉴 ID, 원본 메뉴 주소, 배치 ID, 게시글 수, 다운로드/스킵/실패 수를 확인할 수 있습니다.

## 폴더명 변경

`폴더명 변경`은 저장된 폴더 이름을 앱 안에서 바꾸기 위한 기능입니다.

사용 순서:

1. 왼쪽 목록에서 게시글 또는 메뉴 그룹을 선택합니다.
2. `폴더명 변경`을 누릅니다.
3. 새 폴더명을 입력합니다.
4. 확인 창에서 변경을 승인합니다.

변경 후 앱이 함께 갱신하는 항목:

- 실제 저장 폴더명
- 선택한 게시글의 `meta.json`
- `data/archive_index.json`
- 메뉴 그룹인 경우 `data/batches/{batch_id}.json`의 메뉴 폴더 경로

같은 이름의 폴더가 이미 있으면 변경하지 않고 안내 메시지를 표시합니다.

## 삭제

`저장 항목 삭제`는 선택한 항목의 저장 폴더와 목록 데이터를 함께 삭제합니다.

- 게시글 선택 시: 해당 게시글 폴더와 인덱스 항목을 삭제합니다.
- 메뉴 그룹 선택 시: 메뉴 상위 폴더와 포함된 게시글 인덱스 항목을 삭제합니다.

삭제 전 확인 창이 뜹니다.

## 데이터 파일

- `data/archive_index.json`: 저장된 게시글 목록
- `data/batches/`: 메뉴 일괄 다운로드 결과와 실패 주소 목록
- `data/auth/naver_state.json`: 저장된 브라우저 쿠키/세션 상태
- `data/browser_profile/`: 네이버 로그인 세션이 저장되는 Chromium 프로필
- `saved_posts/`: 게시글 본문, HTML, 이미지, 메타데이터

`data/auth/*.json`과 `data/browser_profile/`은 민감한 세션 정보가 포함될 수 있으므로 git에 올리지 않습니다.

## 문제 해결

### 로그인이 계속 필요하다고 나올 때

1. `네이버 로그인 세션 연결` 버튼을 누릅니다.
2. 열린 Chromium에서 네이버에 직접 로그인합니다.
3. 앱으로 돌아와 세션 상태가 적용되는지 확인합니다.
4. 다시 다운로드를 실행합니다.

### 저장된 글이 비어 있거나 접근 실패가 날 때

현재 로그인한 계정으로 해당 카페 글을 실제로 볼 수 있는지 확인하세요. 카페 가입 여부, 게시판 권한, 멤버 등급 제한이 있으면 앱도 저장할 수 없습니다.

### 다운로드 실패 원인을 확인하고 싶을 때

추출에 실패하면 아래 위치에 디버그 파일이 생성됩니다.

```text
saved_posts/_debug/{timestamp}_{article_id_or_unknown}/
```

생성되는 파일:

- `debug_screenshot.png`
- `debug_page.html`
- `debug_info.json`

### 메뉴 다운로드가 느릴 때

메뉴 다운로드는 게시글을 하나씩 열고 이미지까지 저장합니다. 로그인 세션을 유지하고 요청을 과도하게 보내지 않기 위해 병렬 다운로드를 하지 않습니다. 이미 저장된 게시글은 `article_key`로 스킵되므로 같은 메뉴를 다시 실행하면 처음보다 빠르게 끝납니다.
